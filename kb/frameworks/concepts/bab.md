# BAB

> **Propósito**: Framework de transformação — mostra o contraste entre o estado atual e o estado desejado antes de revelar a ponte.
> **Quando usar**: Depoimentos, abertura de sales pages, conteúdo educacional com oferta, carrossel IG.

## O que é

BAB significa **Antes → Depois → Ponte**. Em vez de começar pela dor (PAS) ou pela atenção (AIDA), o BAB usa o contraste visual entre dois estados para criar desejo. O avatar vê onde está (Antes), imagina onde poderia estar (Depois), e então recebe o caminho para chegar lá (Ponte).

É o framework mais natural para copy de **transformação demonstrada** — depoimentos, cases de alunos, storytelling de resultados. Quando você tem prova concreta de uma virada, BAB é a estrutura que a apresenta com máximo impacto.

No mercado BR de infoprodutos, BAB performa bem em segmentos onde o avatar conhece o resultado desejado mas não acredita que é acessível para ele. A Ponte quebra a dissonância.

## Estrutura

| Etapa | Função | Especificidade recomendada |
|-------|--------|---------------------------|
| **Antes (B)** | Estado atual — dor, limitação, frustração em detalhe | Alta — use números, tempo, sentimento específico |
| **Depois (A)** | Estado desejado — resultado concreto, não abstrato | Alta — imagem vívida, tangível |
| **Ponte (B)** | O mecanismo que liga os dois — produto, método, decisão | Média — suficiente para credibilidade, não para overwhelm |

## Exemplo aplicado (pt-BR, BR-nativo)

**Contexto**: carrossel IG para designer freelancer

> **Antes**: "Slide 1 — Em março de 2023, a Camila cobrava R$ 350 por identidade visual. Levava 2 semanas. Mal pagava o aluguel."
>
> **Depois**: "Slide 2 — Em dezembro do mesmo ano, o ticket mínimo dela era R$ 2.800. Com prazo de 5 dias. E lista de espera de 6 clientes."
>
> **Ponte**: "Slide 3 — A diferença? Ela parou de vender 'design'. Começou a vender posicionamento de marca. O método que usou está no link da bio."

## Quando NÃO usar

- Avatar ainda não sabe que tem problema — use PAS (precisa nomear a dor primeiro)
- Copy de produto técnico/B2B onde o benefício é racional — use FAB (mais preciso)
- VSL longa — use PASTOR (BAB é curto; PASTOR tem resistência e objeção embutidos)
